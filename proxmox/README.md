# Proxmox Virtual Environment

![img](https://encrypted-tbn3.gstatic.com/faviconV2?url=https://pve.proxmox.com&client=AIM&size=128&type=FAVICON&fallback_opts=TYPE,SIZE,URL)

Proxmox — это экосистема решений с открытым кодом для IT-инфраструктуры. **VE** — гипервизор для виртуализации (VM/контейнеры), **Backup Server** — система резервного копирования с дедупликацией, **Mail Gateway** — защита почты от спама/вирусов, а **Datacenter Manager** — централизованная панель для управления множеством независимых кластеров. 

Вот подробное описание каждого компонента:

#### 1. Proxmox Virtual Environment (Proxmox VE)

Это основная платформа (гипервизор), предназначенная для управления виртуальными машинами (KVM) и контейнерами (LXC). 

- **Функции:** Объединяет несколько физических серверов в кластер, обеспечивает высокую доступность (HA), имеет встроенный веб-интерфейс.
- **Для чего:** Создание виртуальных серверов, хранилищ и сетей без необходимости покупать дорогое проприетарное ПО. 

#### 2. Proxmox Backup Server (PBS)

Корпоративное решение для резервного копирования, оптимизированное для работы с Proxmox VE. 

- **Функции:** Инкрементное резервное копирование (бэкап только измененных данных),  полная дедупликация данных (экономия места), быстрое восстановление.
- **Для чего:** Защита данных виртуальных машин и контейнеров от сбоев. 

#### 3. Proxmox Mail Gateway

Антиспам- и антивирусное решение, которое защищает почтовые сервера. 

- **Функции:** Ставится между интернетом и почтовым сервером (Exchange, Postfix и  др.), фильтрует входящую/исходящую почту, блокирует фишинг, вирусы и  спам.
- **Для чего:** Безопасность корпоративной электронной почты. 

#### 4. Proxmox Datacenter Manager (PDM)

Инструмент для централизованного управления, который объединяет несколько независимых кластеров Proxmox VE в одной панели. 

- **Функции:** Мониторинг состояния всех систем, базовые операции управления  (старт/стоп VM), миграция виртуальных машин между независимыми  дата-центрами.
- **Для чего:** Удобное управление масштабной инфраструктурой из одной точки. 

**Итог:** Proxmox VE — ядро, Backup Server — защита, Mail Gateway — безопасность, Datacenter Manager — управление.

---

Proxmox Virtual Environment (VE) — это мощная открытая платформа управления виртуализацией на базе Debian GNU/Linux, объединяющая виртуальные машины (KVM) и контейнеры (LXC). Она  обеспечивает централизованное управление через веб-интерфейс,  поддерживает кластеризацию, живую миграцию и Software-Defined Storage,  что делает её отличным выбором для частных облаков и дата-центров. 

**Основные характеристики Proxmox VE:**

- **Гибридная виртуализация:** KVM для полноценных ВМ (Windows/Linux) и LXC для легких контейнеров.
- **Управление:** Удобный веб-интерфейс (GUI) + командная строка (CLI).
- **Хранилище:** Поддержка локальных директорий, LVM, NFS, iSCSI, Ceph.
- **Кластеризация:** Объединение нескольких узлов в кластер для высокого уровня доступности (HA).
- **Резервное копирование:** Встроенный инструмент Proxmox Backup Server. 

**Системные требования:**

- **CPU:** 64-bit, необходима поддержка аппаратной виртуализации (Intel VT или AMD-V).
- **RAM:** Минимум 1 ГБ (рекомендуется больше для продуктивной работы).
- **Сеть:** Минимально одна сетевая карта. 

Proxmox популярен как в корпоративной среде, так и для домашних лабораторий  из-за своей гибкости, отсутствия лицензионных платежей за функционал и  активного сообщества.

Ошибка «D-sub вне диапазона» (Out of Range) в Proxmox VE 

возникает, когда видеокарта сервера передает разрешение или частоту обновления,  которые монитор (часто старый VGA/D-sub) не поддерживает. Решение — снизить разрешение/частоту в GRUB, используя SSH или безопасный режим, установив параметры `nomodeset` или `vga=791`.

**Основные способы устранения ошибки:**

1. **Подключение по SSH:** Если сеть работает, подключитесь к серверу через SSH с другого компьютера и измените настройки GRUB (см. пункт 2).
2. **Редактирование GRUB (через безопасный режим/recovery):**
   - Перезагрузите сервер Proxmox.
   - В меню загрузчика GRUB выберите "Advanced options for Proxmox VE" -> "Recovery mode".
   - После загрузки введите пароль root.
   - Откройте файл конфигурации: `nano /etc/default/grub`.
   - Найдите строку `GRUB_CMDLINE_LINUX_DEFAULT` и добавьте `nomodeset` или `video=LVDS-1:d` (чтобы отключить неиспользуемый выход), или установите низкое разрешение, например: `GRUB_CMDLINE_LINUX_DEFAULT="quiet nomodeset vga=791"`.
   - Обновите GRUB: `update-grub`.
   - Перезагрузитесь: `reboot`.
3. **Использование другого монитора:** Подключите монитор с более высоким разрешением (HDMI/DisplayPort) для первоначальной настройки.
4. **Проверка кабеля:** Иногда некачественный VGA-кабель передает неправильные данные EDID монитора.

Чаще всего помогает добавление **`nomodeset`** в параметры загрузки, что заставляет систему использовать базовый драйвер VGA.

# Установка Proxmox на чистый Debian

![img](https://encrypted-tbn3.gstatic.com/faviconV2?url=https://pve.proxmox.com&client=AIM&size=128&type=FAVICON&fallback_opts=TYPE,SIZE,URL)

Установка [Proxmox VE 8](https://pve.proxmox.com/wiki/Install_Proxmox_VE_on_Debian_12_Bookworm) на чистый Debian 12 (Bookworm) включает настройку статического IP, обновление системы, добавление репозиториев Proxmox, установку ядра PVE и пакетов. Ключевые шаги: настройка `/etc/hosts`, `apt update && apt dist-upgrade`, добавление ключа репозитория, установка `proxmox-ve` и перезагрузка. 

## Пошаговая установка Proxmox на Debian 12

### 1. Подготовка ОС (Debian 12)

- Установите чистый Debian 12.
- Убедитесь, что у вас настроен статический IP-адрес на роутере.
- Отредактируйте файл `/etc/hosts`, чтобы имя хоста (hostname) было привязано к статическому IP, а не к `127.0.1.1`.

```
# Пример /etc/hosts
127.0.0.1 localhost
192.168.1.10 pve.example.com pve
```

## Установите чистый Debian 12

Для установки чистого Debian 12 (Bookworm) скачайте ISO-образ (рекомендуется `netinst`) с официального сайта, запишите его на флешку через BalenaEtcher/Rufus и загрузитесь с неё. В процессе установки выберите язык, сеть, настройте  пользователей и при выборе ПО отметьте только «Стандартные системные  утилиты» для минимальной установки. 

**Пошаговая установка:**

1. **Подготовка:** Скачайте `amd64 netinst iso` с официального сайта Debian. Запишите образ на USB-накопитель.
2. **Запуск:** Загрузите ПК/сервер с флешки, выберите «Graphical Install» или «Install».
3. **Настройка:** Следуйте шагам установщика (язык, клавиатура, сеть).
4. **Разметка дисков:** Рекомендуется «Авто — использовать весь диск» (или настройте LVM/шифрование по необходимости).
5. **Выбор ПО (Критичный шаг):** На этапе выбора среды рабочего стола (Software Selection) **снимите флажки** со всех графических сред (GNOME, XFCE и т.д.). Оставьте отмеченными только **«Standard system utilities»**, **«SSH»** и **«Apache HTTP server»**.
6. **Завершение:** Установите загрузчик GRUB на основной диск, завершите установку и перезагрузите компьютер.
7. **Результат:** После перезагрузки вы получите чистую консоль Debian 12 без графического интерфейса. 

**Первые шаги после установки:**

- Войдите под `root` или пользователем с `sudo`.
- Обновите систему: `apt update && apt upgrade`.
- Если у вас видеокарта NVIDIA, установите необходимые заголовки ядра: `apt install linux-headers-amd64`. 

### Добавить пользователю `user` права sudo в Debian

В отличие от других дистрибутивов, таких как Ubuntu, **Debian вообще не требует установки sudo** . Для установки sudo необходимо получить права root, используя,  например, команду su, установить пакет sudo, добавить своего  пользователя в группу sudo, затем выйти из системы и снова войти.

Чтобы добавить пользователю `user` права sudo в Debian, нужно добавить его в группу `sudo`. Для этого выполните команду `usermod -aG sudo user` под root-пользователем, затем пользователю потребуется выйти и снова войти в систему для обновления прав. 

**Основные шаги:**

**Войдите под root**: Откройте терминал и переключитесь на суперпользователя:

```
su -
```

**Добавьте пользователя в группу sudo**:

```
usermod -aG sudo user
```

*(Замените `user` на имя нужного пользователя)*.

**Проверьте наличие sudo** (если команда не найдена):

```
apt update && apt install sudo
```


Эта ошибка возникает, так как система пытается использовать установочный  DVD/USB как репозиторий, но он недоступен или не содержит нужных файлов. Чтобы исправить это и установить sudo, нужно отключить CD-ROM в источниках (закомментировать строку) и обновить списки пакетов.

**Решение:**

1. **Откройте файл источников:**
   `nano /etc/apt/sources.list`
2. **Закомментируйте строку с cdrom**, поставив `#` в начале:
   `#deb cdrom:[Debian ...`
3. **Сохраните и выйдете:** (Ctrl+O, Enter, Ctrl+X).
4. **Обновите и установите:**
   `apt update && apt install sudo`

После этого apt будет использовать только сетевые зеркала. 

**Повторяем проверку наличие sudo**:

```
apt update && apt install sudo
```

**Примените изменения**: Пользователь должен перелогиниться (выйти и зайти снова). 

**Альтернативный способ (через `visudo`):**
Если нужно более точное управление правами, можно отредактировать файл sudoers:

1. Выполните `visudo`.
2. Добавьте строку: `user ALL=(ALL:ALL) ALL`. 

После этих действий пользователь сможет выполнять команды, используя `sudo <команда>`.

### Инструкция обновления Debian 12 до Debian 13 

Обновление Debian 12 (Bookworm) до Debian 13 (Trixie) выполняется через редактирование 

`/etc/apt/sources.list`, заменяя "bookworm" на "trixie", с последующим выполнением `apt update` и `apt full-upgrade`. Перед обновлением обязательны создание резервных копий, обновление  текущей системы и очистка старых пакетов для предотвращения конфликтов. 

**Создайте резервную копию:** Перед началом крайне рекомендуется сделать бэкап важных данных.

**Обновите текущую систему (Debian 12):**
Убедитесь, что Debian 12 полностью обновлен:

```
sudo apt update
sudo apt upgrade -y
sudo apt dist-upgrade -y
sudo apt autoremove -y
```

**Измените источники репозиториев (sources.list):**
Замените все упоминания `bookworm` на `trixie` в файле `/etc/apt/sources.list`. Это можно сделать командой:

```
sudo sed -i 's/bookworm/trixie/g' /etc/apt/sources.list
# Если есть файлы в /etc/apt/sources.list.d/, их тоже нужно обновить
sudo sed -i 's/bookworm/trixie/g' /etc/apt/sources.list.d/*.list
```

**Обновите список пакетов и начните обновление:**

```
sudo apt update
sudo apt upgrade -y
```

**Выполните полное обновление (dist-upgrade):**
Это основная команда для перехода на новую версию:

```
sudo apt full-upgrade -y
```

*В процессе могут появляться вопросы о замене конфигурационных файлов.  Обычно рекомендуется оставлять текущие версии или внимательно читать.*

**Очистите систему:**
Удалите устаревшие пакеты:

```
sudo apt autoremove -y
sudo apt clean
```

**Перезагрузка:**

```
sudo reboot
```

Проверка версии:

После перезагрузки проверьте успешность обновления командой:

```
cat /etc/debian_version
# Должно показать версию Debian 13
hostnamectl
```

*Примечание: Если установлена среда KDE, возможны конфликты файлов, которые решаются очисткой пакетов и перезагрузкой.*

## У меня X9DBU-iF материнская плата



На платах **Supermicro X9DBU-iF** порядок дисков в BIOS и в Linux может не совпадать — это нормально. Linux присваивает буквы (`sda`, `sdb`, `sdc`) в зависимости от того, какой контроллер быстрее ответил при загрузке. Главное, что GRUB успешно установился на `/dev/sdb`, значит, система загрузится.

Как подготовить остальные два диска для хранения данных:

В Proxmox диски для данных лучше настраивать через **Web-интерфейс** после завершения установки, но сейчас важно убедиться, что они «чистые».

**1. Посмотрите список всех дисков:**

```
lsblk
```

Вы увидите дерево устройств. Скорее всего, ваши два HDD будут как `sda` и `sdc` (размером по несколько ТБ).

**2. Установите основной пакет Proxmox VE:**
Прежде чем настраивать хранилища, завершите установку самого гипервизора (после перезагрузки с ядром PVE):

```
sudo apt install proxmox-ve postfix open-iscsi -y
```

*(При установке Postfix выбирайте «Local only»)*.

**3. Настройка дисков для резервного хранения (после установки):**
Когда вы зайдете в веб-интерфейс (`https://ваш-ip:8006`), у вас будет два пути для двух HDD:

- **ZFS (Mirror):** Если диски одинакового объема, можно объединить их в «зеркало». Это  даст высокую надежность: если один диск выйдет из строя, данные  сохранятся на втором.
- **LVM или Директория:** Если диски разные, их можно использовать по отдельности (один под бэкапы, другой под ISO-образы или диски ВМ).

Важное замечание по X9DBU-iF:

У этой платы есть встроенный контроллер Intel (черные/синие порты) и иногда дополнительные порты SAS (если есть доп. чип).

- Убедитесь, что в BIOS контроллер стоит в режиме **AHCI**, а не IDE или "RAID" (Intel RST), так как Proxmox (ZFS) лучше работает с дисками напрямую.

**Какой объем у ваших HDD и планируете ли вы объединять их в массив для надежности (RAID1/Mirror)?**



```
user@debian:~$ lsblk

├─sda1   8:1    0 930,6G  0 part /

sdb      8:16   0 232,9G  0 disk 
sdc      8:32   0 149,1G  0 disk 
user@debian:~$ 
```



Судя по выводу `lsblk`, ситуация следующая:

1. **sda (931.5G)** — ваш основной диск, на котором сейчас установлена ОС Debian (точка монтирования `/`).
2. **sdb (232.9G)** — пустой диск (вероятно, тот самый, куда установился GRUB в прошлом шаге).
3. **sdc (149.1G)** — пустой диск.

**Важный нюанс:** Вы устанавливали GRUB на `/dev/sdb`, но сама система сейчас работает с диска `/dev/sda`. На платах **Supermicro X9DBU-iF** это может привести к тому, что после перезагрузки сервер не найдет загрузчик, если в BIOS не выбран именно диск на 232ГБ.

Как поступить с дисками для резервного хранения:

У вас три диска разного объема (1ТБ, 233ГБ, 149ГБ). Объединять их в  зеркальный массив (RAID1) невыгодно, так как вы потеряете много места.

**Лучшая стратегия для Proxmox на этом железе:**

1. **sda (1TB):** Остается под систему и основные виртуальные машины (LVM-Thin).
2. **sdb (233GB):** Можно использовать как отдельное хранилище для **ISO-образов** и временных файлов.
3. **sdc (149GB):** Идеально подойдет под **Бэкапы (резервные копии)** критически важных ВМ.

Что сделать сейчас:

**1. Установите ядро Proxmox (обязательно):**
Без него мы не сможем двигаться дальше.

```

```

Используйте код с осторожностью.

**2. Убедитесь в правильности загрузки:**
Чтобы система точно знала, где искать загрузчик, обновите его еще раз на системном диске:



*(Здесь `192.168.1.143` — ваш статический IP. Убедитесь, что удалили старое имя `debian` из этой строки).*

Выполните команду:

```
hostname --ip-address
```

Она должна вернуть ваш IP: **192.168.1.143**. Если вернет `127.0.0.1`, Proxmox может работать нестабильно.

