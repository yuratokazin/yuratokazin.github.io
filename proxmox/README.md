# Proxmox Virtual Environment

![img](https://encrypted-tbn3.gstatic.com/faviconV2?url=https://pve.proxmox.com&client=AIM&size=128&type=FAVICON&fallback_opts=TYPE,SIZE,URL)

Proxmox — это экосистема решений с открытым кодом для IT-инфраструктуры. **VE** — гипервизор для виртуализации (VM/контейнеры), **Backup Server** — система резервного копирования с дедупликацией, **Mail Gateway** — защита почты от спама/вирусов, а **Datacenter Manager** — централизованная панель для управления множеством независимых кластеров. 

Вот подробное описание каждого компонента:

#### 1. Proxmox Virtual Environment (Proxmox VE)

Это основная платформа (гипервизор), предназначенная для управления виртуальными машинами (KVM) и контейнерами (LXC). 

- **Функции:** Объединяет несколько физических серверов в кластер, обеспечивает высокую доступность (HA), имеет встроенный веб-интерфейс.
- **Для чего:** Создание виртуальных серверов, хранилищ и сетей без необходимости покупать дорогое проприетарное ПО. 

#### 2. Proxmox Backup Server (PBS)

Корпоративное решение для резервного копирования, оптимизированное для работы с Proxmox VE. 

- **Функции:** Инкрементное резервное копирование (бэкап только измененных данных),  полная дедупликация данных (экономия места), быстрое восстановление.
- **Для чего:** Защита данных виртуальных машин и контейнеров от сбоев. 

#### 3. Proxmox Mail Gateway

Антиспам- и антивирусное решение, которое защищает почтовые сервера. 

- **Функции:** Ставится между интернетом и почтовым сервером (Exchange, Postfix и  др.), фильтрует входящую/исходящую почту, блокирует фишинг, вирусы и  спам.
- **Для чего:** Безопасность корпоративной электронной почты. 

#### 4. Proxmox Datacenter Manager (PDM)

Инструмент для централизованного управления, который объединяет несколько независимых кластеров Proxmox VE в одной панели. 

- **Функции:** Мониторинг состояния всех систем, базовые операции управления  (старт/стоп VM), миграция виртуальных машин между независимыми  дата-центрами.
- **Для чего:** Удобное управление масштабной инфраструктурой из одной точки. 

**Итог:** Proxmox VE — ядро, Backup Server — защита, Mail Gateway — безопасность, Datacenter Manager — управление.

---

Proxmox Virtual Environment (VE) — это мощная открытая платформа управления виртуализацией на базе Debian GNU/Linux, объединяющая виртуальные машины (KVM) и контейнеры (LXC). Она  обеспечивает централизованное управление через веб-интерфейс,  поддерживает кластеризацию, живую миграцию и Software-Defined Storage,  что делает её отличным выбором для частных облаков и дата-центров. 

**Основные характеристики Proxmox VE:**

- **Гибридная виртуализация:** KVM для полноценных ВМ (Windows/Linux) и LXC для легких контейнеров.
- **Управление:** Удобный веб-интерфейс (GUI) + командная строка (CLI).
- **Хранилище:** Поддержка локальных директорий, LVM, NFS, iSCSI, Ceph.
- **Кластеризация:** Объединение нескольких узлов в кластер для высокого уровня доступности (HA).
- **Резервное копирование:** Встроенный инструмент Proxmox Backup Server. 

**Системные требования:**

- **CPU:** 64-bit, необходима поддержка аппаратной виртуализации (Intel VT или AMD-V).
- **RAM:** Минимум 1 ГБ (рекомендуется больше для продуктивной работы).
- **Сеть:** Минимально одна сетевая карта. 

Proxmox популярен как в корпоративной среде, так и для домашних лабораторий  из-за своей гибкости, отсутствия лицензионных платежей за функционал и  активного сообщества.

Ошибка «D-sub вне диапазона» (Out of Range) в Proxmox VE 

возникает, когда видеокарта сервера передает разрешение или частоту обновления,  которые монитор (часто старый VGA/D-sub) не поддерживает. Решение — снизить разрешение/частоту в GRUB, используя SSH или безопасный режим, установив параметры `nomodeset` или `vga=791`.

**Основные способы устранения ошибки:**

1. **Подключение по SSH:** Если сеть работает, подключитесь к серверу через SSH с другого компьютера и измените настройки GRUB (см. пункт 2).
2. **Редактирование GRUB (через безопасный режим/recovery):**
   - Перезагрузите сервер Proxmox.
   - В меню загрузчика GRUB выберите "Advanced options for Proxmox VE" -> "Recovery mode".
   - После загрузки введите пароль root.
   - Откройте файл конфигурации: `nano /etc/default/grub`.
   - Найдите строку `GRUB_CMDLINE_LINUX_DEFAULT` и добавьте `nomodeset` или `video=LVDS-1:d` (чтобы отключить неиспользуемый выход), или установите низкое разрешение, например: `GRUB_CMDLINE_LINUX_DEFAULT="quiet nomodeset vga=791"`.
   - Обновите GRUB: `update-grub`.
   - Перезагрузитесь: `reboot`.
3. **Использование другого монитора:** Подключите монитор с более высоким разрешением (HDMI/DisplayPort) для первоначальной настройки.
4. **Проверка кабеля:** Иногда некачественный VGA-кабель передает неправильные данные EDID монитора.

Чаще всего помогает добавление **`nomodeset`** в параметры загрузки, что заставляет систему использовать базовый драйвер VGA.

# Установка Proxmox на чистый Debian

![img](https://encrypted-tbn3.gstatic.com/faviconV2?url=https://pve.proxmox.com&client=AIM&size=128&type=FAVICON&fallback_opts=TYPE,SIZE,URL)

Установка [Proxmox VE 8](https://pve.proxmox.com/wiki/Install_Proxmox_VE_on_Debian_12_Bookworm) на чистый Debian 12 (Bookworm) включает настройку статического IP, обновление системы, добавление репозиториев Proxmox, установку ядра PVE и пакетов. Ключевые шаги: настройка `/etc/hosts`, `apt update && apt dist-upgrade`, добавление ключа репозитория, установка `proxmox-ve` и перезагрузка. 

## Пошаговая установка Proxmox на Debian 12

### 1. Подготовка ОС (Debian 12)

- Установите чистый Debian 12.
- Убедитесь, что у вас настроен статический IP-адрес на роутере.
- Отредактируйте файл `/etc/hosts`, чтобы имя хоста (hostname) было привязано к статическому IP, а не к `127.0.1.1`.

```
# Пример /etc/hosts
127.0.0.1 localhost
192.168.1.10 pve.example.com pve
```

## Установите чистый Debian 12

Для установки чистого Debian 12 (Bookworm) скачайте ISO-образ (рекомендуется `netinst`) с официального сайта, запишите его на флешку через BalenaEtcher/Rufus и загрузитесь с неё. В процессе установки выберите язык, сеть, настройте  пользователей и при выборе ПО отметьте только «Стандартные системные  утилиты» для минимальной установки. 

**Пошаговая установка:**

1. **Подготовка:** Скачайте `amd64 netinst iso` с официального сайта Debian. Запишите образ на USB-накопитель.
2. **Запуск:** Загрузите ПК/сервер с флешки, выберите «Graphical Install» или «Install».
3. **Настройка:** Следуйте шагам установщика (язык, клавиатура, сеть).
4. **Разметка дисков:** Рекомендуется «Авто — использовать весь диск» (или настройте LVM/шифрование по необходимости).
5. **Выбор ПО (Критичный шаг):** На этапе выбора среды рабочего стола (Software Selection) **снимите флажки** со всех графических сред (GNOME, XFCE и т.д.). Оставьте отмеченными только **«Standard system utilities»**, **«SSH»** и **«Apache HTTP server»**.
6. **Завершение:** Установите загрузчик GRUB на основной диск, завершите установку и перезагрузите компьютер.
7. **Результат:** После перезагрузки вы получите чистую консоль Debian 12 без графического интерфейса. 

**Первые шаги после установки:**

- Войдите под `root` или пользователем с `sudo`.
- Обновите систему: `apt update && apt upgrade`.
- Если у вас видеокарта NVIDIA, установите необходимые заголовки ядра: `apt install linux-headers-amd64`. 

### Добавить пользователю `user` права sudo в Debian

В отличие от других дистрибутивов, таких как Ubuntu, **Debian вообще не требует установки sudo** . Для установки sudo необходимо получить права root, используя,  например, команду su, установить пакет sudo, добавить своего  пользователя в группу sudo, затем выйти из системы и снова войти.

Чтобы добавить пользователю `user` права sudo в Debian, нужно добавить его в группу `sudo`. Для этого выполните команду `usermod -aG sudo user` под root-пользователем, затем пользователю потребуется выйти и снова войти в систему для обновления прав. 

**Основные шаги:**

**Войдите под root**: Откройте терминал и переключитесь на суперпользователя:

```
su -
```

**Добавьте пользователя в группу sudo**:

```
usermod -aG sudo user
```

*(Замените `user` на имя нужного пользователя)*.

**Проверьте наличие sudo** (если команда не найдена):

```
apt update && apt install sudo
```

Возникла ошибка:

```
Репозиторий «cdrom://[Debian GNU/Linux 12.7.0 _Bookworm_ - Official amd64 DVD Binary-1 with firmware 20240831-10:40] bookworm Release» не содержит файла Release.
```

Эта ошибка возникает, так как система пытается использовать установочный  DVD/USB как репозиторий, но он недоступен или не содержит нужных файлов. Чтобы исправить это и установить sudo, нужно отключить CD-ROM в источниках (закомментировать строку) и обновить списки пакетов.

**Решение:**

1. **Откройте файл источников:**
   `nano /etc/apt/sources.list`
2. **Закомментируйте строку с cdrom**, поставив `#` в начале:
   `#deb cdrom:[Debian ...`
3. **Сохраните и выйдете:** (Ctrl+O, Enter, Ctrl+X).
4. **Обновите и установите:**
   `apt update && apt install sudo`

После этого apt будет использовать только сетевые зеркала. 

**Повторяем проверку наличие sudo**:

```
apt update && apt install sudo
```

**Примените изменения**: Пользователь должен перелогиниться (выйти и зайти снова). 

**Альтернативный способ (через `visudo`):**
Если нужно более точное управление правами, можно отредактировать файл sudoers:

1. Выполните `visudo`.
2. Добавьте строку: `user ALL=(ALL:ALL) ALL`. 

После этих действий пользователь сможет выполнять команды, используя `sudo <команда>`.

# Установка Proxmox на Debian 13

### Инструкция обновления Debian 12 до Debian 13 

Обновление Debian 12 (Bookworm) до Debian 13 (Trixie) выполняется через редактирование 

`/etc/apt/sources.list`, заменяя "bookworm" на "trixie", с последующим выполнением `apt update` и `apt full-upgrade`. Перед обновлением обязательны создание резервных копий, обновление  текущей системы и очистка старых пакетов для предотвращения конфликтов. 

**Создайте резервную копию:** Перед началом крайне рекомендуется сделать бэкап важных данных.

**Обновите текущую систему (Debian 12):**
Убедитесь, что Debian 12 полностью обновлен:

```
sudo apt update
sudo apt upgrade -y
sudo apt dist-upgrade -y
sudo apt autoremove -y
```

**Измените источники репозиториев (sources.list):**
Замените все упоминания `bookworm` на `trixie` в файле `/etc/apt/sources.list`. Это можно сделать командой:

```
sudo sed -i 's/bookworm/trixie/g' /etc/apt/sources.list
# Если есть файлы в /etc/apt/sources.list.d/, их тоже нужно обновить
sudo sed -i 's/bookworm/trixie/g' /etc/apt/sources.list.d/*.list
```

**Обновите список пакетов и начните обновление:**

```
sudo apt update
sudo apt upgrade -y
```

**Выполните полное обновление (dist-upgrade):**
Это основная команда для перехода на новую версию:

```
sudo apt full-upgrade -y
```

*В процессе могут появляться вопросы о замене конфигурационных файлов.  Обычно рекомендуется оставлять текущие версии или внимательно читать.*

**Очистите систему:**
Удалите устаревшие пакеты:

```
sudo apt autoremove -y
sudo apt clean
```

**Перезагрузка:**

```
sudo reboot
```

Проверка версии:

После перезагрузки проверьте успешность обновления командой:

```
cat /etc/debian_version
# Должно показать версию Debian 13
hostnamectl
```

*Примечание: Если установлена среда KDE, возможны конфликты файлов, которые решаются очисткой пакетов и перезагрузкой.*

### Пошаговая инструкция редактирования файла /etc/hosts:

**Откройте терминал** и отредактируйте файл с правами суперпользователя:

```
sudo nano /etc/hosts
```

**Найдите строку**, начинающуюся с `127.0.1.1`, за которой следует ваше имя хоста (hostname).

**Замените `127.0.1.1`** на ваш статический IP-адрес.
*Пример:*
Было: `127.0.1.1  my-computer`
Стало: `192.168.1.10  my-computer`

```
127.0.0.1       localhost
192.168.1.10    debian

# The following lines are desirable for IPv6 capable hosts
::1             localhost ip6-localhost ip6-loopback
ff02::1         ip6-allnodes
ff02::2         ip6-allrouters
```

**Что не правильно:**

Строка с IP `192.168.1.143` должна быть на новой строке, а не впритык к `127.0.0.1`, такой синтаксис считается ошибкой и может вызвать проблемы с сетевыми службами (например, `sudo` или почтовым сервером).

**Правильный вид:**

```
127.0.0.1       localhost
127.0.1.1       debian
192.168.1.10   debian

# The following lines are desirable for IPv6 capable hosts
::1             localhost ip6-localhost ip6-loopback
ff02::1         ip6-allnodes
ff02::2         ip6-allrouters
```

**Сохраните файл**: Нажмите `Ctrl+O`, затем `Enter`.

**Выйдите из редактора**: Нажмите `Ctrl+X`.

**(Опционально) Обновите имя хоста**, если оно также изменилось, в файле `/etc/hostname`. 

После этого изменения система будет связывать имя хоста с указанным статическим IP-адресом.

Добавьте ключ репозитория для Debian 12:

```
sudo wget https://enterprise.proxmox.com/debian/proxmox-release-bookworm.gpg -O /etc/apt/trusted.gpg.d/proxmox-release-bookworm.gpg
```

Добавьте репозиторий в `/etc/apt/sources.list.d/pve-install.list`:

```
echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" | sudo tee /etc/apt/sources.list.d/pve-install.list
```

Обновите систему:

```
apt update && apt dist-upgrade
```

Готовы переходить к установке **самого пакета Proxmox VE** после перезагрузки?



## У меня X9DBU-iF материнская плата



На платах **Supermicro X9DBU-iF** порядок дисков в BIOS и в Linux может не совпадать — это нормально. Linux присваивает буквы (`sda`, `sdb`, `sdc`) в зависимости от того, какой контроллер быстрее ответил при загрузке. Главное, что GRUB успешно установился на `/dev/sdb`, значит, система загрузится.

Как подготовить остальные два диска для хранения данных:

В Proxmox диски для данных лучше настраивать через **Web-интерфейс** после завершения установки, но сейчас важно убедиться, что они «чистые».

**1. Посмотрите список всех дисков:**

```
lsblk
```

Вы увидите дерево устройств. Скорее всего, ваши два HDD будут как `sda` и `sdc` (размером по несколько ТБ).

**2. Установите основной пакет Proxmox VE:**
Прежде чем настраивать хранилища, завершите установку самого гипервизора (после перезагрузки с ядром PVE):

```
sudo apt install proxmox-ve postfix open-iscsi -y
```

*(При установке Postfix выбирайте «Local only»)*.

**3. Настройка дисков для резервного хранения (после установки):**
Когда вы зайдете в веб-интерфейс (`https://ваш-ip:8006`), у вас будет два пути для двух HDD:

- **ZFS (Mirror):** Если диски одинакового объема, можно объединить их в «зеркало». Это  даст высокую надежность: если один диск выйдет из строя, данные  сохранятся на втором.
- **LVM или Директория:** Если диски разные, их можно использовать по отдельности (один под бэкапы, другой под ISO-образы или диски ВМ).

Важное замечание по X9DBU-iF:

У этой платы есть встроенный контроллер Intel (черные/синие порты) и иногда дополнительные порты SAS (если есть доп. чип).

- Убедитесь, что в BIOS контроллер стоит в режиме **AHCI**, а не IDE или "RAID" (Intel RST), так как Proxmox (ZFS) лучше работает с дисками напрямую.

**Какой объем у ваших HDD и планируете ли вы объединять их в массив для надежности (RAID1/Mirror)?**



```
user@debian:~$ lsblk
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINTS
sda      8:0    0 931,5G  0 disk 
├─sda1   8:1    0 930,6G  0 part /
├─sda2   8:2    0     1K  0 part 
└─sda5   8:5    0   976M  0 part [SWAP]
sdb      8:16   0 232,9G  0 disk 
sdc      8:32   0 149,1G  0 disk 
user@debian:~$ 
```



Судя по выводу `lsblk`, ситуация следующая:

1. **sda (931.5G)** — ваш основной диск, на котором сейчас установлена ОС Debian (точка монтирования `/`).
2. **sdb (232.9G)** — пустой диск (вероятно, тот самый, куда установился GRUB в прошлом шаге).
3. **sdc (149.1G)** — пустой диск.

**Важный нюанс:** Вы устанавливали GRUB на `/dev/sdb`, но сама система сейчас работает с диска `/dev/sda`. На платах **Supermicro X9DBU-iF** это может привести к тому, что после перезагрузки сервер не найдет загрузчик, если в BIOS не выбран именно диск на 232ГБ.

Как поступить с дисками для резервного хранения:

У вас три диска разного объема (1ТБ, 233ГБ, 149ГБ). Объединять их в  зеркальный массив (RAID1) невыгодно, так как вы потеряете много места.

**Лучшая стратегия для Proxmox на этом железе:**

1. **sda (1TB):** Остается под систему и основные виртуальные машины (LVM-Thin).
2. **sdb (233GB):** Можно использовать как отдельное хранилище для **ISO-образов** и временных файлов.
3. **sdc (149GB):** Идеально подойдет под **Бэкапы (резервные копии)** критически важных ВМ.

Что сделать сейчас:

**1. Установите ядро Proxmox (обязательно):**
Без него мы не сможем двигаться дальше.

```
sudo apt install proxmox-default-kernel -y
```

Используйте код с осторожностью.

**2. Убедитесь в правильности загрузки:**
Чтобы система точно знала, где искать загрузчик, обновите его еще раз на системном диске:


```
sudo grub-install /dev/sda
sudo update-grub
```


**3. Перезагрузка:**


```
sudo reboot
```


После перезагрузки, когда введете `uname -a` и увидите **PVE-ядро**, мы установим сам Proxmox VE. А настройку дисков `sdb` и `sdc` мы сделаем уже в красивом **Web-интерфейсе** — это намного проще и надежнее, чем через консоль.

Вы уже определились, какой **статический IP-адрес** будет у этого сервера (например, 192.168.1.100)? Proxmox очень не любит менять IP после установки.





**После перезагрузки** проверьте версию ядра командой `uname -a`. Если там будет упоминание **pve**, можно приступать к установке самого Proxmox VE:


```
sudo apt install proxmox-ve postfix open-iscsi -y
```

У вас уже настроено **статическое имя хоста** в файле `/etc/hosts`? (Proxmox требует, чтобы имя вашего сервера было привязано к его IP-адресу, а не к 127.0.0.1).



# Установка гипервизора Proxmox VE

![img](https://encrypted-tbn3.gstatic.com/faviconV2?url=https://pve.proxmox.com&client=AIM&size=128&type=FAVICON&fallback_opts=TYPE,SIZE,URL)

Теперь переходим к финальной установке самого гипервизора.

#### 1. Установка Proxmox VE

Запустите установку основных пакетов:

```
sudo apt install proxmox-ve postfix open-iscsi -y
```

Далее:

- **Postfix**: При запросе выберите «Local only» (это нужно для системных уведомлений).
- **Удаление старого ядра**: После установки Proxmox предложит удалить стандартное ядро Debian (`6.12.74...`), можно соглашаться.

Инструкция просит выбрать **«Local only»** (Только локальное использование) на первом этапе. Если вы сейчас находитесь в окне настройки `relayhost`, значит, на первом шаге был выбран другой вариант (например, «Internet with smarthost»).

**Что сделать сейчас:**

1. Сотрите `smtp.localdomain`.
2. Оставьте поле **пустым**.
3. Нажмите **Enter**.

**Важное замечание:**
Если вы случайно проскочили выбор «Local only» и выбрали «Internet with  smarthost», Postfix будет пытаться отправить почту наружу, что в  домашней лабе может не сработать без донастройки. После завершения  установки `apt`, вы всегда сможете перенастроить его командой:
`sudo dpkg-reconfigure postfix` — и там уже четко выбрать **«Local only»**, как требует ваша инструкция.

```
sudo dpkg-reconfigure postfix
```

Продолжаем установку **Proxmox**?

#### 2. Настройка доступа (Web-интерфейс)

После завершения установки ваш сервер Proxmox будет доступен по адресу:
**`https://192.168.1.143:8006`**

- **Логин**: root (или ваш текущий пользователь, если у него есть права sudo)
- **Пароль**: Ваш пароль от системы.
- **Важно**: Браузер выдаст предупреждение о «небезопасном соединении»  (SSL-сертификат самоподписанный) — просто нажмите «Дополнительно» ->  «Перейти на сайт».

#### 3. Что делать с дисками sdb и sdc?

Когда зайдете в Web-панель:

1. Выберите в дереве слева ваш сервер (**debian**).
2. Перейдите в раздел **Disks**.
3. Нажмите **Wipe Disk** для `/dev/sdb` и `/dev/sdc`, чтобы полностью очистить их от старых таблиц разделов.
4. Теперь вы можете создать на них хранилища:
   - **Directory** или **LVM** — для хранения ISO-образов или дисков ВМ.
   - **ZFS** — если хотите использовать современные функции снимков (snapshots).

**Кстати**, ваше имя хоста сейчас `debian`. В Proxmox его лучше сменить на что-то уникальное (например, `pve-node1`), так как это имя будет отображаться в кластере.

Установка **proxmox-ve** прошла без ошибок?



Чтобы сменить имя хоста (hostname) на **pve-node1** (или любое другое), нужно отредактировать два системных файла. Proxmox очень чувствителен к этим настройкам.

#### 1. Измените имя в основном конфиге

Замените старое имя `debian` на новое:

```
sudo hostnamectl set-hostname pve-node1
```

#### 2. Отредактируйте файл `/etc/hosts`

Это критически важный шаг. Откройте файл:

```
sudo nano /etc/hosts
```

Найдите строку с вашим IP-адресом (или добавьте её, если её нет). Она должна выглядеть так:

```
127.0.0.1       localhost
192.168.1.143   pve-node1.proxmox.com pve-node1
```

*(Здесь `192.168.1.143` — ваш статический IP. Убедитесь, что удалили старое имя `debian` из этой строки).*

Удалось **увидеть экран логина** Proxmox в браузере?



Чтобы найти **Wipe Disk**, следуйте этой инструкции в веб-интерфейсе:

1. В левой колонке (дерево ресурсов) нажмите на ваш узел **pve-gitcube**.
2. В центральном меню (белый список рядом с деревом) найдите пункт **Disks** (он находится ниже разделов *System* и *Network*).
3. В открывшемся списке вы увидите свои диски `/dev/sdb` и `/dev/sdc`.
4. Нажмите левой кнопкой мыши на нужный диск (например, **sdb**), чтобы он выделился синим.
5. В верхней панели инструментов (над списком дисков) появится кнопка **Wipe Disk**.

Что делать после Wipe Disk:

Когда диск станет полностью «чистым» (статус *unused*), вам нужно создать на нем хранилище, чтобы Proxmox мог его использовать:

- **Для диска 233ГБ (sdb):**
  Перейдите в подраздел **LVM-Thin** (чуть ниже кнопки *Disks* в том же меню) -> Нажмите **Create: Thinpool** -> Выберите диск `sdb` -> Дайте имя (например, `data-ssd`). Здесь будут жить ваши виртуальные машины.
- **Для диска 149ГБ (sdc):**
  Перейдите в подраздел **Directory** -> Нажмите **Create: Directory** -> Выберите диск `sdc` -> Файловая система `ext4` -> Дайте имя (например, `backups`). Сюда вы будете сохранять резервные копии и ISO-образы.

**Важно:** Если кнопка **Wipe Disk** не нажимается (серая), значит на диске есть остатки старых разделов или LVM. В этом случае выделите диск и нажмите **Initialize Disk with GPT** (тоже в верхнем меню), а затем повторите Wipe.

Удалось **очистить диски** и создать первое хранилище?

```
Are you sure you want to wipe /dev/sdb?
All data on the device will be lost!

Type: Hard Disk
Usage: isw_raid_member
Size: 232.88 GiB
Serial: 5VY5WQ3Q
```

Да, вы можете смело нажимать ОК, если на этом диске нет важных данных.

Надпись **Usage: isw_raid_member** означает, что раньше этот диск был частью **Intel Software RAID** (технология Intel Matrix Storage/RST). Поскольку мы устанавливаем  чистый Proxmox, этот старый «рейдовый» заголовок нужно удалить, иначе  система не позволит создать на нем новое хранилище.

Ваши следующие действия:

1. Подтвердите операцию **Wipe** (введите пароль или нажмите «Yes», если спросит).
2. После этого статус диска в списке изменится на **unused** (неиспользуемый).
3. Теперь выберите этот диск в меню **Disks -> LVM-Thin** и нажмите **Create: Thinpool**, чтобы Proxmox мог создавать на нем виртуальные диски для ваших машин.

**Важный момент:** если после Wipe диск всё еще отображается как `isw_raid_member`, может потребоваться нажать кнопку **Initialize Disk with GPT** (там же, в верхнем меню), чтобы окончательно переразметить его под стандарты Proxmox.

```
Are you sure you want to wipe /dev/sdc?
All data on the device will be lost!

Type: unknown
Usage: No
Size: 149.05 GiB
Serial: WD-WCANMH762382
```



Удалось очистить **оба диска** (sdb и sdc) или с каким-то возникли сложности?



Отлично! Теперь, когда диски **sdb** и **sdc** имеют статус **unused**, их нужно «подключить» к Proxmox, чтобы на них можно было хранить образы систем и виртуальные машины.

Что сделать прямо сейчас (в веб-интерфейсе):

#### 1. Подключите диск 233 ГБ (sdb) для виртуальных машин:

- В том же меню **Disks** выберите подраздел **LVM-Thin**.
- Нажмите вверху кнопку **Create: Thinpool**.
- В поле **Disk** выберите `/dev/sdb`.
- В поле **Name** введите, например, `ssd-data`.
- Нажмите **Create**. (Теперь в дереве слева появится новое хранилище для дисков ваших будущих ВМ).

**2. Подключите диск 149 ГБ (sdc) для ISO и бэкапов:**

- В меню **Disks** выберите подраздел **Directory**.
- Нажмите **Create: Directory**.
- В поле **Disk** выберите `/dev/sdc`.
- В поле **Filesystem** оставьте `ext4`.
- В поле **Name** введите `backup-iso`.
- В поле **Content** (содержимое) убедитесь, что выбраны: *VZDump backup file, ISO image, Container template*.
- Нажмите **Create**.

Если поля «Content» нет в окне создания:

1. Просто введите **Name** (например, `backup-iso`), выберите **Disk** (`sdc`) и нажмите **Create**.
2. После того как хранилище появится в колонке слева (под именем сервера `pve-gitcube`), нажмите на него.
3. Перейдите в его настройки: **Edit** (Редактировать) -> вкладка **General**.
4. Там вы увидите список **Content**. Убедитесь, что там выбраны:
   - **ISO image** (для дистрибутивов систем)
   - **VZDump backup file** (для резервных копий)
   - **Container template** (для шаблонов LXC)
   - **Disk image** (если захотите там хранить диски ВМ)

Если кнопки **Edit** (Редактировать) не видно сразу, значит, вы смотрите в раздел настроек самого диска, а нужно смотреть в раздел **Хранилища (Storage)** на уровне всего Датацентра.

Вот как найти настройки **Content** (содержимое):

1. В самом левом столбце (дерево ресурсов) нажмите на самую верхнюю иконку — **Datacenter** (Датацентр).
2. В центральном белом списке выберите пункт **Storage** (Хранилище).
3. Там вы увидите ваш созданный `backup-iso`. Нажмите на него один раз.
4. Сверху появится кнопка **Edit** (Редактировать). Нажмите её.
5. В открывшемся окне найдите выпадающий список **Content** и выделите там (можно выбрать несколько через Ctrl или просто кликами):
   - **ISO image**
   - **VZDump backup file**
   - **Container template**

Как проверить, что всё сработало:

После сохранения загляните снова в левое дерево под сервер `pve-gitcube`. Нажмите на значок диска `backup-iso`. Если в центральном меню появились вкладки **ISO Images** и **CT Templates**, значит, всё настроено верно.

**Кстати**, если вам нужно скачать **ISO-образ** (например, Debian или Ubuntu) прямо на сервер, чтобы не загружать его со своего ПК, в этой же вкладке **ISO Images** есть удобная кнопка **Download from URL**.

Удалось найти настройки **Content** в разделе **Datacenter -> Storage**?



Как загрузить первый ISO-образ:

1. В левом меню нажмите на ваше новое хранилище (например, `backup-iso`).
2. В центральной панели выберите вкладку **ISO Images**.
3. Нажмите кнопку **Upload** (Загрузить) или **Download from URL** (скачать напрямую с сайта, например, Debian или Ubuntu).



Почему это важно:

Теперь у вас есть разделение: основная система Debian на `sda`, быстрые диски виртуалок на `sdb` и отдельное место для скачанных образов `.iso` и резервных копий на `sdc`.


# Бесплатный репозиторий Proxmox VE

![img](https://encrypted-tbn3.gstatic.com/faviconV2?url=https://pve.proxmox.com&client=AIM&size=128&type=FAVICON&fallback_opts=TYPE,SIZE,URL)

*You do not have a valid subscription for this server. Please visit www.proxmox.com to get a list of available options.*

Это стандартное уведомление **Proxmox VE**, которое появляется при каждом входе в веб-интерфейс, если не куплена платная подписка. На работу системы оно **не влияет**.

Чтобы оно не мешало и чтобы сервер корректно обновлялся в домашней лаборатории, нужно сделать две вещи:

#### Отключите платный репозиторий (чтобы `apt update` не выдавал ошибки)

По умолчанию **Proxmox** ищет обновления на серверах для подписчиков. Вам нужно переключить его на бесплатную ветку **No-Subscription**.

В **Proxmox 9** (как и в свежих версиях 8) используется новый формат описания репозиториев — файлы `.sources` вместо привычных `.list`.



### Home Lab на базе Proxmox 9 (Trixie):

#### 1. Удаление платного репозитория

Удаляем файл, который требует подписку и блокирует обновления:

```
sudo rm /etc/apt/sources.list.d/pve-enterprise.sources.
```

#### 2. Добавление бесплатного репозитория (No-Subscription)

Важно указать полный путь `/debian/pve`, иначе система не найдет пакеты:

```
echo "deb http://download.proxmox.com trixie pve-no-subscription" | sudo tee /etc/apt/sources.list.d/pve-no-subscription.list
```

#### 3. Добавление ключа безопасности (Keyring)

Чтобы система доверяла репозиторию Proxmox и не выдавала предупреждений:

```
wget https://enterprise.proxmox.com -O /etc/apt/trusted.gpg.d/proxmox-release-bookworm.gpg
```

*(На текущий момент ключ от Bookworm подходит и для Trixie).*

#### 4. Обновление списков и системы

Теперь всё пройдет без ошибок:

```
sudo apt update && sudo apt dist-upgrade -y
```

**Важный совет:** Не забудьте зайти в раздел **Updates** (Обновления) вашего узла и нажать **Refresh**. Если вылезет ошибка про Enterprise-подписку — это нормально, просто закройте окно, пакеты всё равно обновятся из вашего `no-subscription` репозитория.

Чтобы зайти в раздел Updates в Proxmox VE, откройте веб-интерфейс, выберите конкретный узел (node) в дереве ресурсов слева, перейдите на вкладку «Updates» (Обновления) и нажмите «Refresh» для  обновления списка пакетов. Это позволяет проверять и устанавливать обновления для каждой системы Proxmox и Ceph. 

**Пошаговая инструкция:**

1. **Откройте веб-интерфейс:** Введите в браузере `https://IP-адрес-сервера:8006`.
2. **Выберите узел:** В левой панели «Datacenter» нажмите на имя вашего сервера (узел/node).
3. **Перейдите в раздел:** В верхней панели выберите вкладку **Updates** (Обновления).
4. **Обновите список:** Нажмите кнопку **Refresh** (Обновить) вверху. Proxmox выполнит `apt update`.
5. **Установите обновления:** После обновления списка нажмите **Upgrade** (Обновить) для установки новых пакетов. 

*Примечание: Если у вас нет платной подписки, при нажатии Refresh появится предупреждение, которое нужно просто закрыть (OK).*

Перенос сайта с **Raspberry Pi** на **Proxmox** даст вам гораздо больше ресурсов и надежности. Поскольку вы используете **VPS** (вероятно, как прокси или VPN-туннель), схема переноса будет выглядеть так:

#### 1. Создайте «дом» для сайта (Контейнер LXC)

Для сайта лучше использовать **LXC-контейнер**, а не виртуалку (VM) — он потребляет в 10 раз меньше оперативной памяти.

- В веб-интерфейсе Proxmox нажмите кнопку **Create CT** (справа сверху).
- **Hostname**: например, `web-server`.
- **Template**: Перейдите в ваше хранилище `backup-iso` -> **CT Templates** -> **Templates** и скачайте шаблон (например, `debian-12-standard` или `ubuntu-24.04-standard`).
- **Disks**: Выберите ваш диск `ssd-data` (sdb), укажите размер (например, 20-40 ГБ).
- **Network**: Укажите статический IP (например, `192.168.1.150/24`) и шлюз (`192.168.1.1`).

#### 2. Перенос данных с Raspberry Pi

Самый простой способ перенести файлы и базу данных — через **SCP** или **Rsync** прямо из терминала.

**На вашем новом сервере (в консоли контейнера):**

- Установите веб-стек (Nginx/Apache, PHP, MySQL/MariaDB), такой же как на малинке.

- Скопируйте файлы сайта:

```
scp -r pi@ip-малинки:/var/www/html/* /var/www/html/
```

Сделайте дамп базы данных на малинке и импортируйте его на новый сервер:

```
# На малинке:
mysqldump -u root -p dbname > site_db.sql
# На новом сервере:
scp pi@ip-малинки:~/site_db.sql .
mysql -u root -p dbname < site_db.sql
```

#### 3. Перенастройка VPS (Туннеля)

Так как ваш сайт работал через VPS, вам нужно обновить правила проброса портов или VPN-клиента:

- Если вы использовали **SSH Tunnel** или **Frp/Cloudflare Tunnel**: установите этот же клиент в новый контейнер Proxmox и запустите его.
- Если на VPS стоит **Nginx (Reverse Proxy)** и туннель: измените внутренний IP в конфиге Nginx на VPS, чтобы он смотрел на новый локальный IP контейнера (`192.168.1.150`).

#### 4. Проверка

Отключите Raspberry Pi и проверьте, открывается ли сайт по внешнему адресу.  Благодаря тому, что Proxmox стоит на мощном железе X9DBU-iF, сайт будет  «летать» по сравнению с малинкой.

**Какая следующая задача** в твоем дата-центре: настройка **хранилища** на дисках или создание **сетевого моста** для виртуалок?

У вас уже есть **ISO-образ** системы (например, Windows или Linux), который вы хотите загрузить для создания первой виртуалки?

**Кстати**, вы планируете создавать **виртуальную машину (VM)** (с полноценной ОС типа Windows/Linux) или **контейнер (LXC)** (легкий Linux-сервер для Docker или баз данных)?
