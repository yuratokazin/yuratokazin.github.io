# Proxmox

Proxmox — это экосистема решений с открытым кодом для IT-инфраструктуры. **VE** — гипервизор для виртуализации (VM/контейнеры), **Backup Server** — система резервного копирования с дедупликацией, **Mail Gateway** — защита почты от спама/вирусов, а **Datacenter Manager** — централизованная панель для управления множеством независимых кластеров. 

Вот подробное описание каждого компонента:

#### 1. Proxmox Virtual Environment (Proxmox VE)

Это основная платформа (гипервизор), предназначенная для управления виртуальными машинами (KVM) и контейнерами (LXC). 

- **Функции:** Объединяет несколько физических серверов в кластер, обеспечивает высокую доступность (HA), имеет встроенный веб-интерфейс.
- **Для чего:** Создание виртуальных серверов, хранилищ и сетей без необходимости покупать дорогое проприетарное ПО. 

#### 2. Proxmox Backup Server (PBS)

Корпоративное решение для резервного копирования, оптимизированное для работы с Proxmox VE. 

- **Функции:** Инкрементное резервное копирование (бэкап только измененных данных),  полная дедупликация данных (экономия места), быстрое восстановление.
- **Для чего:** Защита данных виртуальных машин и контейнеров от сбоев. 

#### 3. Proxmox Mail Gateway

Антиспам- и антивирусное решение, которое защищает почтовые сервера. 

- **Функции:** Ставится между интернетом и почтовым сервером (Exchange, Postfix и  др.), фильтрует входящую/исходящую почту, блокирует фишинг, вирусы и  спам.
- **Для чего:** Безопасность корпоративной электронной почты. 

#### 4. Proxmox Datacenter Manager (PDM)

Инструмент для централизованного управления, который объединяет несколько независимых кластеров Proxmox VE в одной панели. 

- **Функции:** Мониторинг состояния всех систем, базовые операции управления  (старт/стоп VM), миграция виртуальных машин между независимыми  дата-центрами.
- **Для чего:** Удобное управление масштабной инфраструктурой из одной точки. 

**Итог:** Proxmox VE — ядро, Backup Server — защита, Mail Gateway — безопасность, Datacenter Manager — управление.

---

Proxmox Virtual Environment (VE) — это мощная открытая платформа управления виртуализацией на базе Debian GNU/Linux, объединяющая виртуальные машины (KVM) и контейнеры (LXC). Она  обеспечивает централизованное управление через веб-интерфейс,  поддерживает кластеризацию, живую миграцию и Software-Defined Storage,  что делает её отличным выбором для частных облаков и дата-центров. 

**Основные характеристики Proxmox VE:**

- **Гибридная виртуализация:** KVM для полноценных ВМ (Windows/Linux) и LXC для легких контейнеров.
- **Управление:** Удобный веб-интерфейс (GUI) + командная строка (CLI).
- **Хранилище:** Поддержка локальных директорий, LVM, NFS, iSCSI, Ceph.
- **Кластеризация:** Объединение нескольких узлов в кластер для высокого уровня доступности (HA).
- **Резервное копирование:** Встроенный инструмент Proxmox Backup Server. 

**Системные требования:**

- **CPU:** 64-bit, необходима поддержка аппаратной виртуализации (Intel VT или AMD-V).
- **RAM:** Минимум 1 ГБ (рекомендуется больше для продуктивной работы).
- **Сеть:** Минимально одна сетевая карта. 

Proxmox популярен как в корпоративной среде, так и для домашних лабораторий  из-за своей гибкости, отсутствия лицензионных платежей за функционал и  активного сообщества.

Ошибка «D-sub вне диапазона» (Out of Range) в Proxmox VE 

возникает, когда видеокарта сервера передает разрешение или частоту обновления,  которые монитор (часто старый VGA/D-sub) не поддерживает. Решение — снизить разрешение/частоту в GRUB, используя SSH или безопасный режим, установив параметры `nomodeset` или `vga=791`.

**Основные способы устранения ошибки:**

1. **Подключение по SSH:** Если сеть работает, подключитесь к серверу через SSH с другого компьютера и измените настройки GRUB (см. пункт 2).
2. **Редактирование GRUB (через безопасный режим/recovery):**
   - Перезагрузите сервер Proxmox.
   - В меню загрузчика GRUB выберите "Advanced options for Proxmox VE" -> "Recovery mode".
   - После загрузки введите пароль root.
   - Откройте файл конфигурации: `nano /etc/default/grub`.
   - Найдите строку `GRUB_CMDLINE_LINUX_DEFAULT` и добавьте `nomodeset` или `video=LVDS-1:d` (чтобы отключить неиспользуемый выход), или установите низкое разрешение, например: `GRUB_CMDLINE_LINUX_DEFAULT="quiet nomodeset vga=791"`.
   - Обновите GRUB: `update-grub`.
   - Перезагрузитесь: `reboot`.
3. **Использование другого монитора:** Подключите монитор с более высоким разрешением (HDMI/DisplayPort) для первоначальной настройки.
4. **Проверка кабеля:** Иногда некачественный VGA-кабель передает неправильные данные EDID монитора.

Чаще всего помогает добавление **`nomodeset`** в параметры загрузки, что заставляет систему использовать базовый драйвер VGA.
